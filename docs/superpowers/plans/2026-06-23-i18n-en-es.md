# EN/ES Language Toggle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a functional English/Spanish toggle to the footer that switches all UI text site-wide, persisted via localStorage, defaulting to English.

**Architecture:** A React context (`LanguageProvider`) stores the active locale and persists it to localStorage. A single translations dictionary (`lib/translations.ts`) maps dotted keys to EN/ES strings. A `useTranslation()` hook provides a `t(key)` function to all client components. Pages that are currently server components will either become client components or extract translatable content into client sub-components.

**Tech Stack:** React 19 context, localStorage, TypeScript

## Global Constraints

- Default locale is always `"en"`, even if browser locale is Spanish
- Blog post content, titles, and dates stay in English (sourced from markdown)
- Technical labels (C++, React, Git, etc.) and proper nouns are not translated
- Footer metadata (coordinates, clock, copyright) stays in English
- `<title>` and `<meta description>` stay in English for SEO
- No URL changes — locale is client-side only
- No `Co-Authored-By` lines in commits
- Tailwind 4 CSS-first config — use existing design tokens
- Follow existing code patterns (font-mono text-[11px] for metadata, font-display for headings, etc.)

```typescript
"use client";

import { createContext, useContext } from "react";

export type Locale = "en" | "es";

export const LanguageContext = createContext<{
  locale: Locale;
  setLocale: (l: Locale) => void;
}>({ locale: "en", setLocale: () => {} });

const translations: Record<Locale, Record<string, string>> = {
  en: {
    // Nav
    "nav.about": "About",
    "nav.projects": "Projects",
    "nav.blog": "Blog",
    "nav.resume": "Resume",
  },
  es: {
    // Nav
    "nav.about": "Sobre mí",
    "nav.projects": "Proyectos",
    "nav.blog": "Blog",
    "nav.resume": "Currículum",
  },
};

export function useTranslation() {
  const { locale, setLocale } = useContext(LanguageContext);
  const t = (key: string): string => translations[locale][key] ?? key;
  return { t, locale, setLocale };
}
```

- [ ] **Step 2: Create `components/LanguageProvider.tsx`**

```typescript
"use client";

import { useState, useEffect, useCallback } from "react";
import { LanguageContext, type Locale } from "@/lib/translations";

export default function LanguageProvider({ children }: { children: React.ReactNode }) {
  const [locale, setLocaleState] = useState<Locale>("en");

  useEffect(() => {
    const stored = localStorage.getItem("locale");
    if (stored === "en" || stored === "es") {
      setLocaleState(stored);
      document.documentElement.lang = stored;
    }
  }, []);

  const setLocale = useCallback((l: Locale) => {
    setLocaleState(l);
    localStorage.setItem("locale", l);
    document.documentElement.lang = l;
  }, []);

  return (
    <LanguageContext value={{ locale, setLocale }}>
      {children}
    </LanguageContext>
  );
}
```
